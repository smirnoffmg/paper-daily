# paper-daily

Раз в день доставляет одну свежую статью по ML/DS/AI в Telegram-канал.

## Назначение

Поддерживать привычку читать по одной научной статье в день, не тратя время на ручной поиск и отбор. Каждый запуск детерминированно выбирает одну статью из пула кандидатов и публикует её в указанный Telegram-канал — вы читаете оттуда с любого устройства, а понравившееся переносите дальше (например, в свой Obsidian-вейл). Состояние нигде не хранится: выбор полностью определяется текущей датой.

## Как это работает

1. **Поиск.** По заданным темам (`PAPER_QUERIES`) тянутся свежие кандидаты из Semantic Scholar Graph API за текущий год, отсортированные по дате публикации.
2. **Фильтр по площадке.** Опционально оставляются только статьи из нужных журналов/конференций (`PAPER_VENUES`) — например, рецензируемых (JMLR, TMLR, TPAMI и т. п.). Пустой список означает «любая площадка».
3. **Ранжирование под ваши интересы.** Кандидаты сортируются по близости к вашему «профилю интересов» — усреднённому эмбеддингу понравившихся статей. Эмбеддинги считаются моделью SPECTER2, заточенной под научные тексты. Если профиль пуст, используется порядок по свежести.
4. **Детерминированный выбор по дате.** Из отранжированного пула выбирается одна статья как функция от текущей даты — без хранения какого-либо состояния. Один и тот же день всегда даёт один и тот же результат, поэтому повторный запуск за тот же день не приведёт к другой статье.
5. **Доставка.** Выбранная статья форматируется и постится в канал через Telegram Bot API.

## Замечание про дедупликацию

Поскольку состояние не хранится, строгой гарантии «никогда не повторить статью» нет. Пул кандидатов меняется день ото дня — выходят новые работы, старые выпадают из выборки за текущий год, — а выбор привязан к дате, а не к истории отправок. На практике это означает, что между днями возможны редкие повторы или пропуски. Это сознательный размен: простота и отсутствие инфраструктуры взамен идемпотентности «ровно один раз».

## Зачем рецензируемые журналы

Топовый ML по большей части публикуется через конференции (NeurIPS, ICML, ICLR, ACL) — они рецензируемые, но идут в proceedings, а не в журналах. Если нужен строго журнальный формат, в `PAPER_VENUES` стоит указывать JMLR, TMLR, TPAMI, Nature Machine Intelligence, Pattern Recognition. Фильтр работает по подстроке в названии площадки без учёта регистра.

## Настройка

Конфигурация — через переменные окружения:

| Переменная             | Назначение                                            | По умолчанию                     |
| ---------------------- | ----------------------------------------------------- | -------------------------------- |
| `TELEGRAM_TOKEN`       | Токен бота от @BotFather (обязательно)                | —                                |
| `TELEGRAM_CHAT_ID`     | `@channelname` или числовой `-100…` (обязательно)     | —                                |
| `PAPER_QUERIES`        | Темы поиска через запятую                             | `machine learning,deep learning` |
| `PAPER_VENUES`         | Допустимые площадки через запятую; пусто = любые      | пусто                            |
| `PROFILE_PATH`         | Файл с понравившимися статьями (по строке на статью)  | `profile.txt`                    |
| `CANDIDATES_PER_QUERY` | Сколько кандидатов тянуть на запрос                   | `100`                            |
| `MODEL_NAME`           | Модель эмбеддингов                                    | `allenai/specter2_base`          |
| `ABSTRACT_MAX_CHARS`   | Обрезка абстракта в посте                             | `800`                            |
| `S2_API_KEY`           | Ключ Semantic Scholar (опционально, поднимает лимиты) | —                                |

Бот должен быть добавлен администратором в канал, иначе постить он не сможет.

## Профиль интересов

Один раз положите в `profile.txt` 10–20 статей, которые вам нравились, — по одной на строку, в формате `Заголовок. Абстракт` (или просто заголовок). Этот набор задаёт направление ранжирования; чем он репрезентативнее, тем точнее подбор.

## Запуск

Скрипт не хранит состояния и рассчитан на запуск по расписанию (cron, systemd-timer или GitHub Actions). Поскольку выбор полностью определяется датой, ничего коммитить обратно в репозиторий не нужно — запуск идемпотентен в пределах одного дня.

```bash
python -m paper_daily
```
